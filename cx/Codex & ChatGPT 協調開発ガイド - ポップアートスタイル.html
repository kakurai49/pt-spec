Title: Implement deterministic Python core (G_alg) for automated test design from common test model (M)

## Background
We separate the pipeline into:
- Probabilistic extraction (LLM): spec/logs â†’ common test model M (out of scope for this issue)
- Deterministic algorithms (Python): M â†’ test conditions A(S) and optimized test suite (this issue)

We want a Python library + CLI that:
1) Loads/validates a common test model (M) from JSON
2) Generates test conditions A(S) for multiple techniques (EP/BVA/DecisionTable/Pairwise/StateTransition)
3) Produces candidate tests and evaluates coverage
4) Selects a minimal/risk-aware suite via greedy set cover
5) Exports results as JSON (MVP), optionally CSV/pytest later

## Goals
- Deterministic output: same input JSON â‡’ same outputs (stable ordering)
- Extensible architecture: generator plugins per technique
- Safe predicate evaluation (NO python eval)

## Non-goals (MVP)
- No actual LLM calls
- No full constraint solver (Z3) required in MVP
- No integration with external test management tools yet

## Inputs
A single `model.json` representing M. Provide Pydantic schema or dataclass validation.

### Model JSON (MVP schema)
- parameters: list of {name, type, domain, boundary_candidates?}
- rules: list of {id, when, then, description?}
- state_machines: list of {name, states[], transitions[]}
- nfrs: list of {id, metric, comparator, threshold}
- risks: list of {id, severity, note, targets?}
- scenarios: list of {id, persona?, goal?, steps[]}
- observations: list of {id, metric/log, slo?}

### Expression DSL
Use a safe JSON-based expression DSL (JsonLogic-like).
Example:
{"and":[{"<=":[{"var":"age"},0]},{"==":[{"var":"country"},"JP"]}]}

## Outputs
- `conditions.json`: list[TestCondition]
- `suite.json`: list[TestCase] with coverage mapping

### TestCondition shape
- id: str
- technique: str
- description: str
- predicate: JSON (expression DSL)
- covers: list[str]   # coverage item ids
- risk_weight: float? # optional

### TestCase shape
- id: str
- inputs: object (paramâ†’value) OR trace (for state-machine paths)
- covers: list[str]  # condition ids covered
- cost: float? (default 1.0)

## Architecture / Modules
Create package `testgen/`:

testgen/
  model.py            # Pydantic models + validation
  expr.py             # expression DSL evaluator (safe)
  directives.py       # Directive (S)
  conditions.py       # TestCondition, CoverageItem
  candidates.py       # Candidate representation + producer(s)
  generators/
    base.py           # ConditionGenerator interface
    bva.py            # boundary value analysis generator
    ep.py             # equivalence partitioning generator (basic)
    decision_table.py # decision table generator (basic)
    pairwise.py       # pairwise/t-wise generator (MVP: pairwise only)
    state.py          # state transition coverage generator (MVP: state+transition)
  evaluator.py        # evaluate candidate vs condition predicates
  optimizer.py        # greedy set cover
  export.py           # exporters (json)
  cli.py              # CLI entrypoint

## CLI
Command: `python -m testgen --model model.json --out out/ --techniques bva,ep,dt,pairwise,state`

Expected files:
- out/conditions.json
- out/suite.json

## Acceptance criteria
- [ ] `python -m testgen --help` works
- [ ] Loading invalid model JSON fails with clear error messages
- [ ] Each generator returns deterministic, stable-ordered conditions
- [ ] Greedy optimizer returns a suite that covers all conditions (unless impossible)
- [ ] Unit tests cover: expr evaluator, each generator, optimizer
- [ ] No use of `eval`, `exec`, or unsafe AST evaluation

## Verification steps (must pass)
- `python -m pytest -q`
- `python -m testgen --model examples/min_model.json --out /tmp/out --techniques bva,ep`

## Example fixture
Create `examples/min_model.json` with:
- numeric param with min/max and thresholds
- enum param
- at least 2 boolean conditions for decision table
- simple state machine with 2 states, 1 transition
Then ensure conditions/suite are generated.

Please add AGENTS.md with repo conventions:
- Python 3.11+
- black/ruff/mypy settings (if you add them, wire into tests)
- deterministic ordering rules (sort keys, stable ids)
