ゴール

ゴール（メモの要件を具体化）

  • Pythonトレーニングに合わせて、タロット占いのコードを設計・実装する
  • フルスタックで利用できるように、バックエンドAPIとフロントエンドUIを用意する
  • 設計は「同値化（等価クラス化）で世界を離散化する」考え方で整理する
  • テストは DTD（Diffusion Test Design）観点で、観点束 S と A(S) を作って網羅する
  • ドキュメントは4ページ（本サイト）で提供する
補足：このコースは「占いアプリを作りながら、仕様化・設計・API・UI・テストの基本を学ぶ」ことが目的です。 占いの正しさを保証するものではありません（娯楽用途）。
成果物

成果物の全体像

  • タロットデッキデータ：カード78枚（大アルカナ22＋小アルカナ56）をJSON/辞書で管理
  • 占いロジック：スプレッド（1枚/3枚/拡張）・正逆位置・意味生成
  • API：/api/deck, /api/draw 等（入力検証・seed指定で再現可能）
  • Web UI：質問→スプレッド選択→ドロー→結果表示
  • テスト：同値分割＋境界値＋デシジョンテーブル＋性質テスト（任意）

学習ステップ

おすすめ学習ステップ（1〜2週間想定）

  1. データモデル：カードを辞書/JSONで表現（型・バリデーション）
  2. 抽選：重複なしで N枚引く。seedで再現できるようにする
  3. 解釈：正位置/逆位置＋スプレッド位置（過去/現在/未来）でテキスト生成
  4. CLI：対話式に占いを実行（入出力、例外）
  5. API：FastAPIでJSON返却（入力検証、HTTP）
  6. フロント：fetchでAPI呼び出し、結果表示
  7. テスト：観点（S）→等価クラス（A(S)）→テスト生成
このドキュメントは「メモ書き（写真）」の指示に基づき、4ページ構成（overview / implement_spec / issues / spec）で生成されています。
注意：占い結果は娯楽目的です。医療・法律・投資などの重要判断には使用しないでください。
