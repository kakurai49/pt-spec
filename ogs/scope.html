OGS (Obligation Gate System)

Scope / 到達イメージ

PULL型の運用とオブリゲーション（a1〜a12）を軸に、品質合格を再現可能にするトレーニングコース。 運用設計・監査性・品質判断の分離を学び、GateBoardとJudge OKまで一連の流れを理解します。

Scope Index Overview Spec
学習のゴール
期間の目安1日1時間 × 10日
対象ロールmember / leader / admin / reviewer
主なテーマPULL運用・obligation設計・監査性
アウトプットGateBoardとJudge OKの運用像
実装ではなく「運用設計」と「品質合格の可視化」にフォーカスしたコースです。
成果物イメージ
  • Obligation Catalog（a1〜a12）とDoD・証跡要件の整理。
  • キャンペーン単位のObligationItem生成とQueue運用フロー。
  • EvidenceとAudit Eventsを用いた監査可能なログ構造。
  • GateBoard / Evaluation / Judge OKの可視化と最終判断の記録。
最終的に「誰が・いつ・何を・どの義務で合格させたか」を説明できる状態が到達点です。
ロール分離の学び

member

キューからObligationItemをclaimし、実行結果とEvidenceを提出する。

leader / admin

キャンペーン作成・obligation初期化・Judge OKの確定を担う。

reviewer（任意）

Evidenceと結果のレビュー承認で品質判断を補強する。

実行者と判断者を分離することで、PULL実行と品質判断が混ざらない状態を作ります。
対象外 / 非ゴール
  • 実装言語の詳細チューニング（API実装の最適化など）。
  • 確率的に揺れるテストや、運に依存した合格判定。
  • テスト技法の網羅よりも、obligationベースの合格義務を優先。
