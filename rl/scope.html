SIRIUS (Reinforcement Learning)

Scope / 到達イメージ

Bandit → MDP → Q-learning を 14日プランで動かしながら、RLの基本とGitHub運用を重ねて体験するコース。 Scopeでは「どこまで作るか」と「動く姿」を最短で共有します。

期間の目安1日1時間 × 14日
環境ブラウザでデモ確認（外部CDNなし） / GitHubでPR提出
観点探索・活用のバランス、状態遷移、価値推定

「動く仕様書」で腕ごとの報酬・後悔を可視化し、Issue / PR / テストで学習ログを残す前提です。
到達する「動く姿」
  • Bandit（K腕・p[i]可変）をブラウザで操作し、アルゴリズム別の報酬/後悔/選択回数を確認できる。
  • MDPの小さな例（状態・行動・遷移・報酬）が文章で定義され、Issueに落とし込める。
  • Q-learningの更新式とハイパーパラメータが、テスト観点（S, A(S)）と紐づいた形で説明される。
  • GitHub運用：Issueで課題提示 → PRで提出 → テストで採点、を想定した導線が全ページに存在。
Scope（範囲）

Bandit

K腕・p[i]入力・seed再現・ランダム / ε-greedy / UCB の切替。

  • 累積報酬・累積後悔のプロット
  • 腕ごとの選択回数と推定値を表示
  • シード入力でGitHub上の議論を再現

MDP

状態・行動・遷移確率の最小サンプルを提示し、価値の更新を文章で整備。

  • 4〜6状態のミニ環境を想定
  • 報酬設計とエピソードの流れを表で整理
  • 後続のQ-learning課題への橋渡し

Q-learning

更新式、学習率、割引率、εスケジュールの説明を、テスト観点と紐づける。

  • 状態-行動価値表の例と初期化方針
  • エピソード数・探索率の推奨レンジ
  • 「正しく減衰しているか」をチェックするテスト観点
