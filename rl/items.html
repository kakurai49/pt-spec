学習項目 / Training Issues

受講者向けのトレーニングIssueを「学習項目」として集約しました。Bandit → MDP → Q-learning の順に、 seed固定・非flakyを前提とした提出物の粒度を確認できます。

Scope Index 学習項目 Overview Spec
共通ルール
  • seed / ステップ数 / アルゴリズムを提出物に明記し、再現できる形で共有する。
  • 平均regretや成功率のような統計値で判定し、1テイクの偶然に依存しない。
  • 1 Issue = 1テーマ。A_j（影響obligation）が分かるように、観点と証拠を対応づける。
ログ追加は禁止。再現手順と期待値ベースの判定で、非flakyな提出を優先してください。
ミッション別 学習項目
Mission 01: Bandit
  • p配列とseedを指定して ε-greedy / random / UCB を比較。累積報酬・期待後悔を表で共有。
  • 腕ごとの選択回数・推定値をスクリーンショットまたは表で残す。
  • 再現手順：seed / steps / ε または c（UCB）をIssue本文に記載。
Mission 02: GridWorld / DP
  • 決定論的GridWorld（4〜6状態）の遷移表と報酬設計を文章・表で提示。
  • 価値反復 or 方策反復を1ステップ示し、収束イメージを説明。
  • 再現手順：状態集合 / 行動集合 / 遷移確率 / 割引率 / 反復回数を明記。
Mission 03: TD / Q-learning
  • 学習率・割引率・εスケジュールを2パターン試し、平均報酬または成功率を比較。
  • 学習済みQからgreedy policyを導出し、固定seedで success_rate ≥ 0.8 を目安に評価。
  • 再現手順：episodes / seeds / 評価エピソード数 / 評価指標をIssue本文に記載。
各ミッションのアウトプットは docs/rl 配下の観点と結びつけ、A_j を列挙してレビューを軽量化してください。

提出前セルフチェック
  • seedを変えても傾向が変わらないか（最低2 seedで確認）。
  • Banditは期待後悔、Q-learningは success_rate / avg_return のように統計値で比較しているか。
  • Issueタイトル・本文に学習条件を含め、PR本文で再掲できるようにしているか。
