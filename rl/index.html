Scope Index Overview Spec
GitHub運用（標準フロー）
  1. Issueを読む：課題のScope/期待アウトプットを把握し、疑問はコメントで整理。
  2. ブランチを切る：小さなステップでPRを積み上げ、1PR=1テーマを意識。
  3. テストを走らせる：Banditデモはブラウザで、Q-learning以降はユニットテスト（任意）で観点を確認。
  4. PRを出す：試行回数・所要時間・得られた報酬/後悔の変化を簡潔に記録。
  5. レビュー＆再挑戦：フィードバックをIssueに反映し、seed付きで再現できる状態を保つ。
Banditデモは seed / パラメータ付きで共有できるので、PR説明に実行条件を必ず残してください。

Mission（プレースホルダ）
Mission 01 自分のp配列でBanditを回し、ε-greedyとrandomを比較。seedと結果をIssueに貼る。
Mission 02 MDPの状態遷移表を作り、報酬設計の意図を短く説明。PRで図や表を共有。
Mission 03 Q-learningの学習率・割引率を2パターン試し、エピソード報酬の推移をまとめる。
ミッションはリポジトリのIssue化を想定。PRテンプレに「seed」「アルゴリズム」「ステップ数」を記録するだけで再現性が保てます。
